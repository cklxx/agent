# 日志系统优化总结

## 📋 优化目标

根据您的要求，对debug日志进行精简，重点输出以下核心内容：
- **大模型相关的输出**：LLM调用、规划生成、响应解析
- **Agent规划的输出**：任务分解、步骤规划、执行状态

## 🔧 主要改进

### 1. 创建了新的日志配置系统

**文件：`src/config/logging_config.py`**
- 提供了精简和调试两种模式
- 统一管理日志级别配置
- 专门针对LLM和Agent模块优化

### 2. 精简了日志格式

**精简模式（默认）：**
- 去掉了时间戳和模块名，使用纯消息格式
- 重点突出关键信息，使用emoji增强可读性
- 隐藏第三方库的冗余日志输出

**调试模式：**
- 保留详细的时间戳和模块信息
- 显示完整的执行过程和参数

### 4. 第三方库日志控制

将以下模块的日志级别调整为ERROR/WARNING：
- `httpx`, `requests`, `urllib3` (HTTP请求库)
- `langchain`, `langgraph` (LangChain框架)
- `openai._base_client`, `httpcore` (OpenAI客户端)

## 📊 日志级别配置

### 精简模式（默认）
```python
# 核心模块 - INFO级别
"src.agents", "src.graph.nodes", "src.llms"

# 重要的LLM日志 - INFO级别  
"code_agent_llm_planner", "code_agent_llm_execution", "llm_planner"

# 第三方库 - ERROR级别（基本隐藏）
"httpx", "requests", "langchain", "langgraph"
```

### 调试模式
```python
# 核心模块 - DEBUG级别
"src.agents", "src.graph.nodes", "src.llms"

# LLM日志 - DEBUG级别
"code_agent_llm_planner", "code_agent_llm_execution", "llm_planner"

# 第三方库 - WARNING级别（显示重要信息）
"httpx", "requests", "langchain", "langgraph"
```

## 🚀 使用方法

### 命令行
```bash
# 精简模式（默认）
python main.py "您的查询"

# 调试模式
python main.py --debug "您的查询"
```

### 代码中
```python
from src.workflow import run_agent_workflow_async

# 精简模式
await run_agent_workflow_async(user_input="查询", debug=False)

# 调试模式  
await run_agent_workflow_async(user_input="查询", debug=True)
```
